# Claude 디자인 요청 문구 (복붙용)

아래 전체를 Claude(디자인/아티팩트)에 그대로 붙여넣으세요. 현재 사이트 스크린샷 2~3장(전체 화면, 사이드바, 확대해서 마커 보이는 화면)을 함께 첨부하면 결과가 훨씬 좋아집니다.

---

## (여기부터 복사)

원신(Genshin Impact) 인터랙티브 지도 웹앱의 **UI/UX와 색 테마 전체를 리디자인**해줘. 기능은 이미 완성돼 있고, **디자인(CSS와 필요한 HTML 구조/클래스)** 만 새로 만들고 싶어. 결과물은 실제 코드에 바로 적용할 거라, 아래 **기존 구조/클래스명을 그대로 유지**해줘.

### 1. 제품이 뭔지
- 겐신 팬 지도 사이트(“원신맵스” 스타일). 큰 지도 위에 눈동자·보물상자·특산물·광물 등 **마커 수천 개**가 카테고리별로 표시됨.
- 사용자는 지도를 탐색하고, 마커를 클릭해 **“먹었음(수집 완료)”** 체크하면 지도에서 사라짐. 상단에 “수집 X / Y” 진행도 표시.
- 왼쪽 사이드바에서 **카테고리 그룹별 필터**(켜기/끄기), 상단에서 **지도 선택**(티바트 대륙, 연하궁, 서리달 등)과 **로그인**.
- 데스크톱·모바일 둘 다 씀(폰으로 공략 보면서 체크하는 경우 많음).

### 2. 반드시 지킬 기술 제약 (중요)
- **바닐라 HTML/CSS/JS + Leaflet.js 지도 라이브러리** 기반의 정적 사이트. React/프레임워크/빌드 도구 없음.
- 지도 영역(`#map`)은 Leaflet이 그리므로 **건드리지 말 것**(그 안의 타일/마커 위치는 JS가 제어). 지도를 감싸는 레이아웃과 그 위에 얹는 UI만 디자인.
- **기존 element id / class 이름을 그대로 유지**해줘(아래 인벤토리 참고). 새 요소를 추가해도 되지만 기존 훅은 남겨둬야 JS가 동작함.
- 외부 폰트/아이콘 CDN을 써도 되지만, **핵심 동작은 폰트 없이도 깨지지 않게**.
- 마커가 많아 성능이 중요 — 무거운 그림자/블러 남발 금지, 애니메이션은 절제.

### 3. 디자인 방향
- **원신 게임 UI 감성**을 현대적 웹으로: 게임 인게임 UI는 밝은 양피지/크림 바탕 + **골드 테두리·장식**, 그리고 원소별 포인트 컬러(바람=청록, 불=주황red, 물=파랑, 번개=보라, 얼음=하늘, 바위=노랑, 풀=연두). 이 감성을 **깔끔하고 과하지 않게** 웹 UI로 번역해줘.
- **라이트 테마(양피지/크림+골드)와 다크 테마(짙은 남색/그레이+골드·청록 포인트) 둘 다** 제공하고, `prefers-color-scheme` 자동 + 토글 버튼. 지도가 어두운 편이라 다크 테마가 특히 중요.
- 톤: 고급스럽고 판타지스럽되 정보 밀도가 높아도 **읽기 쉽게**. 여백·타이포·위계가 분명하게.
- 지도가 주인공이라 UI는 지도를 가리지 않는 **반투명 패널/떠 있는 카드** 스타일이 잘 어울림.

### 4. 화면 구조와 컴포넌트별 요구 (기존 클래스/ID 유지)

**상단 바 `#topbar`** — 로고/제목(`h1` “내 원신 맵”), 지도 선택 `<select id="mapSelect">`, 진행도 `#stats`(예 “수집 12 / 565”), 동기화 상태 `#syncStatus`, 로그인 영역 `#authArea`(`#loginBtn`, 펼침 폼 `#loginForm`〔`#loginEmail`,`#loginSend`〕, 로그인 후 `#userBox`〔`#userEmail`,`#logoutBtn`〕), 관리자 버튼 `#adminToggle`/`#exportBtn`/`#clearCustomBtn`.
- 지도 선택은 이 앱의 핵심 내비게이션이라 **눈에 잘 띄게**(단순 셀렉트보다 예쁜 드롭다운 느낌 환영).
- 진행도 `#stats`는 **진행 바/뱃지**처럼 시각화하면 좋음.

**사이드바 `#sidebar`** — 상단에 “먹은 것 숨기기” 토글 `#hideCompletedRow`(안에 `#hideCompleted` 체크박스), 버튼 `.sidebar-actions`(`#showAllBtn` 전체 켜기 / `#hideAllBtn` 전체 끄기), 제목 `h2`, 필터 목록 `#filters`, 안내문 `.tip`.
- 필터 목록 구조: 그룹 `.grp`(재생성 자원 그룹은 `.grp.renew`) → 헤더 `.grp-head`(펼침 화살표 `.caret`, 그룹 체크박스, 그룹명 `.grp-title`, 개수 `.count`) → 펼치면 `.grp-body` 안에 항목들 `.filter-row`(체크박스 + 카테고리 아이콘 이미지 `.cat-icon` 또는 색 점 `.dot` + 이름 + 개수 `.count`).
- 이 필터 영역이 **가장 자주 쓰는 부분** — 스캔하기 쉽고, 켜짐/꺼짐 상태가 명확하고, 그룹 접기/펼치기가 자연스럽게. 카테고리 아이콘(작은 이미지)이 잘 어울리게.
- 모바일에선 사이드바가 **접히거나 하단 시트/드로어**로 바뀌면 좋음.

**지도 위 요소들** — 확대 유도 힌트 `#zoomHint`(“🔍 확대하면 마커가 표시됩니다”, 지도 상단 중앙 떠 있는 pill), Leaflet 기본 줌 컨트롤(`.leaflet-control-zoom`)도 테마에 맞게 스타일.

**마커 배지(지도 위 핀)** — 클래스 `.mk` > `.mk-badge`(원형, 카테고리 색 테두리 `border-color` 인라인 지정됨) 안에 카테고리 아이콘 `<img>`(또는 색 점 `.mk-dot`). 지도 배경이 다양해도 **잘 보이게**(적당한 배경/테두리/그림자). 완료된 마커는 흐리게(현재 opacity로 처리). 핀 모양(물방울형) 또는 원형 배지 등 예쁜 형태 제안 환영.

**마커 팝업**(Leaflet 팝업, `.leaflet-popup-content-wrapper` 등) — 내부 구조: 제목 `.popup-title`(아이콘 `.popup-icon` + 이름), 카테고리 `.popup-cat`, 설명 `.popup-desc`, 액션 `.popup-actions`(수집 버튼 `.done-btn`〔완료 시 `.is-done`〕, 삭제 `.del-btn`, 재생성 안내 `.renew-note`). 작은 카드처럼 예쁘게.

**관리자 패널 `#adminPanel`** — 마커 추가 폼(제목 `h3`, `label` 들, `#adminCategory` 셀렉트, `#adminName`/`#adminDesc` 입력). 우하단 떠 있는 카드.

**로그인 폼 `#loginForm`** — 상단바에서 펼쳐지는 작은 팝오버(이메일 입력 + 버튼 + 안내문 `.tip`).

**공통 요소** — 버튼/셀렉트/인풋/체크박스의 통일된 스타일, 스크롤바, 포커스 링(접근성), 로딩/토스트(선택).

### 5. 인터랙션·반응형·접근성
- 반응형: 데스크톱(사이드바 고정) / 태블릿 / 모바일(사이드바 드로어, 큰 터치 타깃). 지도는 항상 최대 영역.
- 접근성: 명확한 대비(WCAG AA), 키보드 포커스 표시, 체크박스/토글 상태가 색뿐 아니라 형태로도 구분.
- 마이크로 인터랙션은 절제(호버/체크 트랜지션 정도). 과한 애니메이션 금지.

### 6. 원하는 산출물
1. **CSS 테마 파일 한 벌**(현재 `css/style.css`를 대체할 수 있는 형태). CSS 변수로 라이트/다크 팔레트 정의, 컴포넌트별 스타일 포함.
2. 라이트/다크 **토글 UI**와 그 동작에 필요한 최소 HTML/JS 스니펫.
3. 필요하면 상단바·사이드바 등 **개선된 HTML 구조 제안**(단, 위의 id/class 훅은 유지하고, 추가 래퍼/요소만 더하는 방식).
4. **색 팔레트·타이포·간격 스케일**을 표로 정리.
5. 데스크톱/모바일 **목업**(HTML/CSS 아티팩트로 실제 렌더되는 예시)이면 최고.

먼저 전체 무드와 팔레트, 컴포넌트 스타일 방향을 제안하고, 그다음 실제 CSS로 구현해줘. 요소 id/class는 반드시 위 인벤토리대로 유지해줘.
